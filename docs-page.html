Installation

Ready to unleash the full potential of your website with WebWiz? Our documentation is here to guide you through the setup process, introduce you to key features, and provide step-by-step instructions for customizing your website to perfection.

Prerequisites 2.1

Before installing WebWiz on your localhost, make sure you have the following prerequisites installed on your system:

  • Nodejs

    Make sure to have Node.js installed & running on your computer. If you already have installed Node on your computer, you can skip this step if your existing node version is greater than 18. We suggest you to use LTS version of Node.js.

  • Npm

    Make sure to have the Npm installed & running on your computer. If you already have installed Npm on your computer, you can skip this step. We suggest you use Npm instead of NPM.

  • Mongodb

    Make sure to have the Mongodb installed & running on your computer. If you already have installed gulp on run command npm install -g mongodb from your terminal.

  • Git

    Make sure to have Git installed globally & running on your computer. If you already have installed git on your computer, you can skip this step.

Setup Instructions 2.2

Upon purchasing WebWiz, you've taken the first step toward creating a dynamic and customizable website. In this guide, we'll walk you through the process of setting up your MERN stack project, including extracting the files, understanding the folder structure, and running the necessary commands to start and build your project.

1. Downloading and Extracting the Files:

After downloading the WebWiz ZIP file, locate the file in your downloads folder or designated directory. Extract the ZIP file to reveal the contents. Inside, you'll find several folders, including "template," "rest-api," and "dashboard."

2. Understanding the Folder Structure:
  • Template: This folder contains the frontend template files for your website. It includes React components, stylesheets, and other frontend assets. You'll primarily work with this folder when customizing the appearance and user interface of your website.
  • Rest API:

    This folder houses the backend files for your project. It includes the server-side code written in Node.js, Express.js, and MongoDB for handling data storage and retrieval. You'll interact with this folder when managing your website's data and implementing server-side logic.

  • Dashboard:

    The dashboard folder contains the administrative interface for managing your website. It includes React components, stylesheets, and other frontend assets specific to the admin dashboard. You'll utilize this folder to manage website content, settings, and user permissions.

NPM Commands Reference
Command Description
npm install This command installs all the required dependencies listed in the project's package.json file into the node_modules folder. It ensures that all necessary packages and modules are available for the project to run smoothly.
npm start Use npm start to start the development server. This command will compile your project's source code, start the server, and launch your application in the default web browser. It's the primary command used to run your project locally during development.
npm run build This command is used to create a production-ready build of your project. It compiles and optimizes your source code, bundles all assets, and prepares the application for deployment. The output is typically stored in a build or dist folder.
npm test Use npm test to execute tests defined in your project. This command runs any test suites or scripts specified in the package.json file. It's commonly used for automated testing and ensuring the reliability of your codebase.
